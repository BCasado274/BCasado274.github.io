AWS

AWS es un servicio...

🔹 Configuración del Entorno y Seguridad en AWS

Para asegurar un entorno en la nube bien configurado y protegido, seguí estas buenas prácticas de seguridad, redes y monitoreo en AWS:

🔹 1. Seguridad de la Cuenta

  • Activación de MFA: Implementé la autenticación multifactor (MFA) para agregar una capa adicional de seguridad.
  • Gestión de usuarios y permisos: Configuré AWS Identity and Access Management (IAM) para crear usuarios, grupos y roles con privilegios mínimos necesarios.

🔹 2. Configuración de la Red

  • Creación de una VPC: Definí una Virtual Private Cloud (VPC) para aislar recursos dentro de una red segura.
  • Configuración de subredes y acceso: Implementé subredes, gateways de internet y tablas de rutas para controlar el tráfico de red.

🔹 3. Implementación de Buenas Prácticas de Seguridad

  • Grupos de Seguridad y NACLs: Configuré reglas en los Security Groups y Network ACLs para restringir el acceso a instancias y bases de datos.
  • Cifrado de Datos: Utilicé AWS Key Management Service (KMS) para gestionar claves de cifrado.

🔹 4. Monitoreo y Gestión del Entorno

  • Amazon CloudWatch: Implementé métricas y alertas para monitorear el rendimiento y la actividad de los recursos.
  • AWS CloudTrail: Activé auditoría de eventos para rastrear cambios y accesos a la cuenta.

🔹 5. Planificación de Continuidad del Negocio

  • Respaldo y Recuperación: Configuré soluciones de respaldo en Amazon S3 y AWS Backup para proteger datos críticos.

🔹 6. Optimización de Costos

  • Monitoreo de Gastos: Utilicé AWS Cost Management Console para analizar costos y definir presupuestos.
  • Uso eficiente de recursos: Implementé estrategias como instancias reservadas y autoescalado para optimizar costos.

Reto 1: Crear y configurar una cuenta en AWS

Este reto consistió en crear mi cuenta en uno de los servicios de nube más populares: Amazon Web Services (AWS). A continuación, detallo el proceso de registro, la configuración del entorno, y las alertas de presupuesto que añadí para evitar sorpresas debido a posibles cargos por el uso de herramientas de pago.

Proceso de Creación de Cuenta: La creación de una cuenta en AWS comienza en su página principal, donde el usuario debe registrar una nueva cuenta ingresando su dirección de correo electrónico, eligiendo un nombre de usuario y proporcionando información personal y de facturación. Al ser una cuenta de prueba, seleccioné la opción de usar la capa gratuita de AWS, que ofrece acceso a muchos servicios sin costo durante el primer año.

Configuración del Entorno: Una vez creada la cuenta, el siguiente paso fue configurar el entorno de trabajo. Para ello, se accedió al AWS Management Console, donde se pueden gestionar todos los servicios de AWS. Es importante que, dado que AWS ofrece una variedad de herramientas de pago, se añadió una alerta de presupuesto en el servicio AWS Budgets para evitar cargos accidentales. Esta alerta avisa cuando se acerca al límite de uso gratuito o si se incurre en algún cargo inesperado.

Algunas de las Herramientas Más Usadas: Algunos de los servicios más utilizados de AWS que exploré incluyen:

  • CLI de AWS: La interfaz de línea de comandos (CLI) permite interactuar con los servicios de AWS desde la terminal, facilitando la automatización de tareas y el manejo de recursos de manera eficiente.
  • EC2 (Elastic Compute Cloud): EC2 es uno de los servicios más emblemáticos de AWS. Permite crear instancias de servidores virtuales para ejecutar aplicaciones, sitios web y servicios, todo con un control total sobre el sistema operativo y las configuraciones.
  • S3 (Simple Storage Service): S3 es el servicio de almacenamiento de objetos, utilizado principalmente para almacenar grandes volúmenes de datos como archivos, imágenes, y copias de seguridad de manera segura y accesible desde cualquier lugar.

Reto 1: Crear y configurar una cuenta en GCP

Este reto consistió en crear cuentas en uno de los servicios de nube más populares: Amazon Web Services (AWS) y Google Cloud Platform (GCP). A continuación, detallo el proceso para cada uno de los servicios, la configuración del entorno, y las alertas de presupuesto que añadí para evitar sorpresas debido a posibles cargos por el uso de herramientas de pago.

Proceso de Creación de Cuenta: El proceso para crear una cuenta en Google Cloud es similar al de AWS. En primer lugar, el usuario debe registrarse utilizando una cuenta de Google y proporcionar información de facturación. Al igual que en AWS, se ofrece una capa gratuita con créditos para usar en los primeros 90 días, lo que permite explorar y probar muchos servicios sin coste adicional.

Configuración del Entorno: Tras completar el registro y la creación de la cuenta, se accedió al Google Cloud Console, el cual sirve para gestionar y configurar los diferentes servicios de la nube de Google. En este caso, también configuré una alerta de presupuesto en Google Cloud Billing para evitar el uso accidental de servicios de pago y mantener los costos dentro de los límites establecidos.

Algunas de las Herramientas Más Usadas: Algunos de los servicios más utilizados de GCP que exploré incluyen:

  • Compute Engine: Este servicio permite crear máquinas virtuales (VM) para ejecutar aplicaciones, servicios web y cualquier otro tipo de carga de trabajo. Es uno de los servicios más versátiles para gestionar infraestructuras en la nube.
  • Google Cloud Storage: Al igual que en AWS, GCP ofrece un servicio de almacenamiento de objetos con alta disponibilidad y escalabilidad para almacenar datos, copias de seguridad y archivos de grandes volúmenes.
  • Cloud SDK: La herramienta de línea de comandos de Google Cloud (gcloud) permite administrar los recursos de GCP desde la terminal. Es especialmente útil para realizar tareas de automatización y manejo de recursos sin necesidad de acceder constantemente a la consola web.

About

Tras completar el bachillerato, estudié una carrera en Marketing, pero mi verdadero interés por la tecnología surgió cuando tuve la oportunidad de desarrollar un videojuego junto a un grupo de amigos. Aquel proyecto despertó mi pasión por la programación, lo que me llevó a especializarme en el área a través de un FP en Desarrollo de Aplicaciones Web. En el último año, descubrí el mundo del Cloud Computing y decidí que quería enfocarme en esta tecnología. Actualmente, estoy ampliando mis conocimientos en Sistemas Cloud (GCP, AWS y Azure) mediante un curso de Fundae, con el objetivo de obtener la certificación Google Cloud Engineer Associate.

En cuanto a mi stack tecnológico, tengo experiencia con Java, SQL, PHP, HTML, CSS, JavaScript, Bash en Linux, Git y Python (este último en un nivel más básico, enfocado en análisis de datos). Me motiva el aprendizaje constante y la aplicación de nuevas tecnologías para desarrollar soluciones optimizadas a problemas reales. Creo en la importancia de mejorar cada día tanto a nivel personal como profesional. Además de la programación, disfruto jugando al ajedrez, ya que potencia el pensamiento lógico y abstracto. También soy entrenador de pádel, lo que me ha ayudado a desarrollar habilidades de liderazgo y trabajo en equipo.
