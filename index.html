Reto 1: Crear y configurar una cuenta en AWS

Este reto consistió en crear cuentas en uno de los servicios de nube más populares: Amazon Web Services (AWS) y Google Cloud Platform (GCP). A continuación, detallo el proceso para cada uno de los servicios, la configuración del entorno, y las alertas de presupuesto que añadí para evitar sorpresas debido a posibles cargos por el uso de herramientas de pago.

Proceso de Creación de Cuenta: La creación de una cuenta en AWS comienza en su página principal, donde el usuario debe registrar una nueva cuenta ingresando su dirección de correo electrónico, eligiendo un nombre de usuario y proporcionando información personal y de facturación. Al ser una cuenta de prueba, seleccioné la opción de usar la capa gratuita de AWS, que ofrece acceso a muchos servicios sin costo durante el primer año.

Configuración del Entorno: Una vez creada la cuenta, el siguiente paso fue configurar el entorno de trabajo. Para ello, se accedió al AWS Management Console, donde se pueden gestionar todos los servicios de AWS. Es importante que, dado que AWS ofrece una variedad de herramientas de pago, se añadió una alerta de presupuesto en el servicio AWS Budgets para evitar cargos accidentales. Esta alerta avisa cuando se acerca al límite de uso gratuito o si se incurre en algún cargo inesperado.

Algunas de las Herramientas Más Usadas: Algunos de los servicios más utilizados de AWS que exploré incluyen:

  • CLI de AWS: La interfaz de línea de comandos (CLI) permite interactuar con los servicios de AWS desde la terminal, facilitando la automatización de tareas y el manejo de recursos de manera eficiente.
  • EC2 (Elastic Compute Cloud): EC2 es uno de los servicios más emblemáticos de AWS. Permite crear instancias de servidores virtuales para ejecutar aplicaciones, sitios web y servicios, todo con un control total sobre el sistema operativo y las configuraciones.
  • S3 (Simple Storage Service): S3 es el servicio de almacenamiento de objetos, utilizado principalmente para almacenar grandes volúmenes de datos como archivos, imágenes, y copias de seguridad de manera segura y accesible desde cualquier lugar.

About

Tras completar el bachillerato, estudié una carrera en Marketing, pero mi verdadero interés por la tecnología surgió cuando tuve la oportunidad de desarrollar un videojuego junto a un grupo de amigos. Aquel proyecto despertó mi pasión por la programación, lo que me llevó a especializarme en el área a través de un FP en Desarrollo de Aplicaciones Web. En el último año, descubrí el mundo del Cloud Computing y decidí que quería enfocarme en esta tecnología. Actualmente, estoy ampliando mis conocimientos en Sistemas Cloud (GCP, AWS y Azure) mediante un curso de Fundae, con el objetivo de obtener la certificación Google Cloud Engineer Associate.

En cuanto a mi stack tecnológico, tengo experiencia con Java, SQL, PHP, HTML, CSS, JavaScript, Bash en Linux, Git y Python (este último en un nivel más básico, enfocado en análisis de datos). Me motiva el aprendizaje constante y la aplicación de nuevas tecnologías para desarrollar soluciones optimizadas a problemas reales. Creo en la importancia de mejorar cada día tanto a nivel personal como profesional. Además de la programación, disfruto jugando al ajedrez, ya que potencia el pensamiento lógico y abstracto. También soy entrenador de pádel, lo que me ha ayudado a desarrollar habilidades de liderazgo y trabajo en equipo.
